# Git 學習指南：專案開發的「時光機」

這份文件記錄了我們關於 Git 的教學內容與後續步驟，方便你隨時查閱。

---

## 1. 核心觀念：為什麼需要 Git？

Git 是開發者的「存檔與時光機」。當你與 AI（Antigravity）協作時，這尤其重要：

*   **安心存檔 (Commit)**：每完成一個功能，就做一次存檔。
*   **出錯還原 (Revert)**：萬一改爛了，可以一秒回到上一動，不怕程式碼毀掉。
*   **備份與同步**：可以上傳到 GitHub，換電腦也能抓回所有開發紀錄。

---

## 2. 常用基礎指令 (Cheat Sheet)

| 指令 | 用途 | 什麼時候用？ |
| :--- | :--- | :--- |
| `git init` | **初始化** | 在新專案開始時執行一次。 |
| `git add <檔案>` | **暫存** | 把修改好的檔案移到「存檔預備區」。常用 `git add .` 加入全部。 |
| `git commit -m "說明"` | **提交存檔** | 撰寫這次改動的內容並正式存檔。 |
| `git status` | **檢查狀態** | 看看哪些檔案改了但還沒存檔。 |
| `git log` | **檢視紀錄** | 查看過去所有的存檔歷史。 |
| `git checkout .` | **還原改動** | 拋棄目前所有「未存檔」的改動，回到上一次存檔狀態。 |

---

## 3. 與 Antigravity 協作的最佳實踐

1.  **大改前先 Commit**：叫我大重構前，你自己先 `git commit` 一次。
2.  **善用計畫功能**：我會先給你 `implementation_plan.md`，你確認後我才會動手。
3.  **異動追蹤**：你可以隨時用 `git diff` 看看我剛才到底改了哪幾行程式碼。
