# 公主連結 Re:Dive — 世界觀與遊戲機制設定指南 (pcrd_universe.md)

本文件是本專案中所有 AI 智能體理解《公主連結 Re:Dive》台服遊戲世界觀與底層開發機制的官方權威指南。所有專有名詞與時程設定均以專案內的繁體中文資料庫及 JSON 文本為準。

---

## 一、 世界觀設定 (World Settings)

### 1. 背面世界與虛實邊界
《公主連結 Re:Dive》第三部的世界觀中，除了主角們原先身處的虛擬世界「阿斯特萊亞」之外，還拓展到了「背面世界」的三個國家。總共劃分為四個主要世界與一個其他/聯動世界：

*   **阿斯特萊亞（Astraea）**：
    *   本作劇情的起點與主體世界。原為一款由超先進人工智慧「米奈娃（Minerva）」運作的虛擬實境網路遊戲（Astrum / 阿斯特魯姆），玩家需要佩戴名為 **mimi** 的 VR 頭戴式裝置登入。
    *   **再啟動（Re:Dive）**：由於霸瞳皇帝強行奪取了米奈娃的世界底層控制權限並重置了世界，導致所有玩家喪失現實世界的記憶，靈魂與意識被強行禁錮在阿斯特萊亞世界中無法登出（斷線崩潰）。
    *   **索爾之塔（Tower of Sol）**：位於世界中心的巨塔，傳說中通往塔頂便能打通與現實世界的連結、讓所有人安全斷線登出。
*   **吉歐‧提格尼亞（Geo Theogonia）**：
    *   第三部主線登場的背面世界地底國度之一。一個建築與植物都充滿甜點與糖果風格、和平且多元種族共存的秩序國度。
    *   主要相關人物包括大天使公主 **萊拉耶爾（Lailael）**、庫露露、克蘿茜，以及「幻變少女」公會的莉莉、普蕾西亞、可璃亞。
*   **吉歐‧格黑納（Geo Gehenna）**：
    *   第三部登場的背面世界地底國度之一。一個由妖魔人統治、魔能充沛且環境嚴酷的煉獄國度，崇尚弱肉強食的魔物世界。
    *   主要代表人物為該國公主 **涅妃＝涅羅（Nefi-Nera）**，以及安涅默涅、拿娜、阿剌克涅、鳳凰、華音。
*   **吉歐‧尼布爾黑爾（Geo Nibelheim）**：
    *   第三部登場的背面世界地底國度之一。一個幽暗無光、由各地代表共同治理的永夜寒冷世界。
    *   主要代表人物為公主 **薇歐莉特（Violet）**，以及志那都、雪野、剎鬼、格蕾斯。
*   **其他世界 / 聯動合作（Other）**：
    *   指因時空扭曲或特別活動，由其他作品前來阿斯特萊亞世界作客的異世界旅行者。
    *   主要代表人物包括亞里莎、吉塔、安、古蕾婭、雷姆、拉姆、愛蜜莉雅、艾姬多娜、班比、露娜、碧卡拉、源櫻等。

### 2. 核心權能與勢力
*   **七冠（Seven Crowns）**：開發這款虛擬實境遊戲的七位創始人，在虛擬世界中擁有掌握底層規則的極高權限：
    *   **霸瞳皇帝（千里真那）**：第一部最終 Boss。其持有的權能為【霸瞳天星】（千里眼，能看穿一切資料與底層程式碼）。
    *   **迷宮女王（模索路晶）**：主角群的引路人，行會【拉比林斯】會長。權能為【迷宮女王】（創造與改造物件，等同於程式修改與熱插拔）。
    *   **誓約女君（克莉絲提娜）**：權能為【絕對迴避】與【絕對命中】。
    *   **副教授（似似花）**：變裝與複製的專家。
    *   **跳躍王（拉吉拉吉）**：空間跳躍大師。
*   **公主騎士（Princess Knight）**：七冠賜予其守護者的特殊能力，能強化行會成員。佑樹（騎士君）為拉比林斯達的公主騎士。


## 二、 遊戲機制與資料庫邏輯 (Game Mechanisms)

### 1. 戰隊戰 (Clan Battle) 運作機制
戰隊戰是行會共同挑戰 Boss 的定番活動。在資料庫設計與邏輯上，須特別注意：
* **階段（Phase）劃分**：
  * **第一階段 (Phase 1)**：周目 (Lap) 為 1。
  * **第二階段 (Phase 2)**：周目 (Lap) 2 ~ 5。
  * **第三階段 (Phase 3)**：周目 (Lap) 6 ~ 10。
  * **第四階段 (Phase 4)**：周目 (Lap) 11 ~ 34。
  * **第五階段 (Phase 5)**：周目 (Lap) 35 及以上。
  * *註：在不同的階段下，Boss 的血量（HP）、防禦力、攻擊力與狂暴機制會有急遽變化，資料會記錄於 `clan_battle_2_map_data` 中。*
* **時程與 ID 錯位**：
  * 資料庫中 `clan_battle_schedule` 包含多種系統測試資料（如 ID 9999 等）。
  * 國服/台服的正式戰隊戰 ID 通常在大於 1000 且小於 2000 的區間。
  * 查詢最新戰隊戰時，不應使用模糊的月份推算，應精確讀取該期活動設定行中（如 ID `1096`）的戰隊戰開始與結束時間。

### 2. 雙端劇情路由與相容性 (TW Database Gotchas)
* **單一明文表路由**：
  * **日版 (JP)** 資料庫中，主線、公會、個人、活動劇情分別存放在 `story_detail`、`chara_story_detail`、`guild_story_detail` 等不同的子表中。
  * **台版 (TW)** 資料庫 `redive_tw.db` 中，**上述所有子表皆不存在**。所有劇情明細均被統一存放在單一明文表 **`story_detail`** 中。
  * **台版分流 ID 規則**：
    * 個人劇情：`story_id >= 1000000 AND story_id < 2000000`
    * 主線劇情：`story_id >= 2000000 AND story_id < 3000000`
    * 公會劇情：`story_id >= 3000000 AND story_id < 4000000`
    * 系統/其他劇情：`story_id >= 4000000 AND story_id < 5000000`
    * 活動劇情：讀取自去混淆後的 `event_story_detail` 表。
  * 任何去混淆或查詢邏輯必須依據 `isTW` 動態切換此路由規則，防止在台版模式下因表不存在而報錯。
