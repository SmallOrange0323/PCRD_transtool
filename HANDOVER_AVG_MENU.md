## 實作功能

### 1. 遊戲化主選單與懸停看板 (MainMenu & Hover Visual)
- **目錄大廳**：實作了 2x2 毛玻璃質感的遊戲卡片式選單，分為「主要劇情」、「角色劇情」、「公會劇情」、「額外劇情」四大版塊。
- **懸停 CG 看板**：左側區域設置了動態看板大背景。當滑鼠懸停在不同卡片上時，會自動切換為相關的主題 CG（如：美食殿堂、貪吃佩可與凱留、公會日常等）。
- **麵包屑路由**：在進入各子分類後，畫面上方支援「🏠 劇情大廳 / [當前分類]」的雙向麵包屑導航，隨時可一鍵返回主目錄。

### 2. 對話流文內對白 CG 圖片與背景
- **文內 CG 內嵌**：在對話歷史串流中，若解析到 `item.type === 'still'` 節點，會自動將對應的 CG 大圖或活動 CG（自動處理 Fallback 備用網址）以白色圓角陰影框內嵌於兩句台詞氣泡框之間，高度還原傳統小說插圖 CG 體驗。
- **場景切換指示**：當解析到 `item.type === 'background'` 節點時，會在對話流中渲染一個虛線的「🎬 場景切換：[背景ID]」指示條，並在載入對白時，自動將該話的第一張場景背景設置為左側看板的大背景。

### 3. 台版資料庫與分類路由相容性修正
- **台版資料路由**：由於台版資料庫 `redive_tw.db` 僅存在一個統一的明文劇情表 `story_detail`，原有日版模式下查詢 `chara_story_detail` 等表會直接報錯並導致列表空白。
  - 我們實作了 `isTW` 動態判斷。
  - 在台版模式下，所有劇情表查詢（個人、公會、主線、系統）皆被自動路由至 `story_detail` 表，並透過 `story_id` 區間進行分流（個人 `1M~2M`，主線 `2M~3M`，公會 `3M~4M`，系統與露娜塔 `4M~5M`），彻底修補了台版模式下列表為空的問題。
- **日文簡稱與譯名映射**：在 `getCharaRealName` 中新增對照表（例如 `ジュ -> 純`，`シュ -> 雪`，`アオ -> 碧`，`カオ -> 香織`，`ユカ -> 由加莉`，`コッ/コッコロ -> 可可蘿` 等），避免對白頭像或登場名單顯示日文簡寫，並在統計中過濾了旁白、系統與店員等非真實角色。

### 4. 資源更新
- 已執行 `download_stories_tw.py` 重新下載並還原了 **8,949 話** 對話 JSON。現在全部 JSON 已含有 `type`、`still`、`background` 標記。

## 本地部署與編譯打包

1. **編譯打包**：
   執行以下命令，會自動將更新後的核心檔案部署至 `dist_story_map/` 目錄：
   ```bash
   python dashboard/scripts/bundle_story_map.py
   ```
2. **啟動伺服器**：
   ```bash
   python -m http.server 8085
   ```
3. **實機驗收網址**：
   - 劇情地圖主頁：[http://localhost:8085/dashboard/story_map.html](http://localhost:8085/dashboard/story_map.html)
   - 獨立打包版：[http://localhost:8085/dist_story_map/index.html](http://localhost:8085/dist_story_map/index.html)
